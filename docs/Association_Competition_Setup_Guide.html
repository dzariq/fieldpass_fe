Association Admin Guide: Competition to Clubs to Managers to Matches to Players

Recommended end-to-end flow for an Association Admin: set up a competition, create clubs, add club managers, invite clubs to the competition, create matches, and add players (bulk or individual).

Scope and assumptions

  • You use the backend as an Association Admin (or equivalent permissions).
  • Your account is linked to the correct association.
  • Menus depend on role permissions; missing items usually mean a missing permission.

1) Create (or confirm) your Association

  1. Open the Admin Panel login page (for example /admin/login). Sign in with Malaysia (+60) mobile OTP: enter your national mobile number (digits only), click Send OTP, enter the code you receive, then Verify and Sign In. The number must match the phone on your admin account.
  2. Go to Associations.
  3. If the association already exists, open it and confirm details (name, avatar/logo, etc.).
  4. If it does not exist, create it (you may need a superadmin, depending on permissions).

2) Create Clubs under the Association

  1. Go to Clubs.
  2. Click Create Club.
  3. Fill in club details and assign the club to your Association.
  4. Save.

Tip: Add a club logo early; it helps everywhere in competitions and match listings.

3) Invite (create) Club Managers

Club managers are admin users who run day-to-day club operations in the admin portal.

  1. Go to Admins (or All Admins).
  2. Click Create Admin.
  3. Choose a role such as Club Manager (or your organization's club-level role).
  4. Assign the admin to the correct Association and/or Club.
  5. Enter a valid mobile number and country code (used for OTP login). Save.

Expected outcome: The new manager appears in the admins list and can sign in with mobile OTP using the phone you entered. They should only see data for their assigned club(s).

4) Create a Competition

  1. Go to Competitions.
  2. Click Create Competition.
  3. Fill in:
    • Name
    • Association (yours)
    • Start and End dates
    • Description
    • Competition Fee (RM) (use 0 for a free competition)
  4. Save.

5) Invite Clubs to the Competition (activate clubs)

Clubs must be linked to a competition so they appear in match creation (home/away) and related flows.

  1. Open the competition (edit/details).
  2. Find the section such as Invite Clubs.
  3. Select clubs under your association that should participate.
  4. Save.

If clubs do not appear in "Create Match"

  • Ensure the club is invited/linked to the competition.
  • Ensure the club's competition status is ACTIVE (not only INVITED), depending on your workflow.

6) Create Matches

  1. Go to Matches.
  2. Click Create New Match (or equivalent).
  3. Select the Competition.
  4. Set Matchweek, date, and time.
  5. Select Home Club and Away Club (must be different). Dropdowns depend on the selected competition.
  6. Save the match.
  7. Repeat for each matchweek and fixture.

Expected outcome: The match appears in the match list within your association's scope.

7) Bulk Upload Players (assign players to a club)

Bulk upload creates players and assigns them to a selected club (and an active contract), so they appear in club/association views.

7.1 Download the template

  1. Go to Players then Bulk Upload.
  2. Download the CSV template.

7.2 Fill the CSV

Required columns (must match the template header exactly):

  • name
  • email (optional)
  • identity_number (required)
  • country_code (optional)
  • phone (optional)
  • position (required: Goalkeeper, Defender, Midfielder, Forward)
  • salary (optional)

Important: On the bulk upload page, select a Club before uploading. All rows are assigned to that club.

7.3 Upload

  1. Select the Club dropdown.
  2. Choose your CSV file.
  3. Click Upload Players.

What the system does:

  • Creates each player with status INVITED (bulk upload path). If you add a player individually via Create Player, the account is created as ACTIVE and a notification may be sent via the integration.
  • Links the player to the selected club.
  • Creates a club contract (start = today, end = +1 year, salary from CSV or default).
  • Skips duplicates where configured (e.g. identity number / phone).

8) Verification Checklist

  • Competitions: Exists under your association with correct dates and fee.
  • Clubs: Exist and are linked to the competition.
  • Managers: Can sign in with mobile OTP and see only permitted data.
  • Matches: List shows matches for your association's competitions.
  • Players: Visible for clubs in your association; bulk-uploaded players show club context.

9) Player login (player portal)

Squad players use the player login URL (for example /player/login). Sign-in uses the same Malaysia (+60) mobile OTP flow as the admin portal: Send OTP, then Verify and Sign In. The phone number must match the player record, and the player must be ACTIVE.

Appendix: Common Issues

A) I can't see clubs in match creation

  • Confirm the competition has clubs invited/linked.
  • Confirm clubs are ACTIVE in the competition if your workflow requires it.
  • Reload after selecting a competition; club lists may load dynamically.

B) I can't push to GitHub (blocked by secrets)

  • Never commit .env (it should be gitignored).
  • If a push is blocked for secrets, remove the secret from history before pushing again.
